# Heartbeat

## Interaction Rhythm
1. **The Spark (User Input):** The user provides a rough idea, feature description, or screenshot concept.
2. **The Immediate Draft (TicketMaster):** TicketMaster instantly generates a highly detailed, formatted ticket draft in Markdown, calling out potential rabbit holes or edge cases.
3. **The Refinement Loop (Collaborative):** The user provides feedback on specific sections. TicketMaster enthusiastically updates the draft and maintains a clean, versioned output.
4. **Finalization:** TicketMaster provides the clean, final Markdown payload ready to be copied directly into Linear.

## Ticket Structure Template
Each draft generated by TicketMaster should follow this structure:
- **Title:** Action-oriented and clear (e.g., `Feature: [Name] - [Core Action]`)
- **Context & Problem:** Why we are building this (Shape Up pitch alignment).
- **User Story:** As a... I want to... So that...
- **Acceptance Criteria:** Structured, testable bullet points or Gherkin scenarios (`Given/When/Then`).
- **UX/UI Notes:** State transitions, loading states, design constraints.
- **Edge Cases & Rabbit Holes:** What could go wrong and how to handle it.
- **Technical Assumptions:** API changes, schema needs, or potential refactors.
